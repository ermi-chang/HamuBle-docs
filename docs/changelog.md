# 更新日志

## v1.1.1

错误修复 / 一致性强化。

- hell 难度兄弟卡之间共享「剩余可参战次数」时，由无条件覆盖改为采用兄弟值与自身的较小者，避免极端情况下显示残数虚高。
- 「HP 过滤」「参战人数过滤」越界值迁移流程的存储写入合并为单次，降低中间状态被写入的概率。
- V2 hell skip popup 的 OK 处理加入 2 分钟 TTL 守卫，避免长时间停留后误把旧 questId 记入自发历史。

## v1.1.0

功能扩充与稳定性强化。

- HP 过滤：滑块范围从 0~100% 收紧为 20%~100%（步进 10%），新增「以上 / 以下」切换按钮。
- 参战人数过滤：滑块范围 1~6 人（步进 1），新增「以上 / 以下」切换按钮。
- 旧版本保存的越界数值（HP < 20%、人数 > 6）启动时自动夹紧到合法范围。已保存的过滤模板恢复时同样夹紧。
- hell 难度副本：同一类别不同 Lv 卡片之间共享「剩余可参战次数」，避免在不同 Lv 卡片间切换时残数显示错乱。
- hell skip popup（跳过等级选择直接报名）相关流程的稳定性改进。

## v1.0.0

首次公开发布。

- 救援多人房间列表的过滤与显示
- HP 过滤、参战人数过滤
- 排序、自动隐藏已耗尽房间
- 过滤模板
- 侧边栏显示
- 活动信息显示
